## 즉시 개선 필요 (Critical)

### ✅ 1. 네비게이션 구조 혼란 - **해결 완료**

**문제점**:

- 사이드바에 표시되는 메뉴와 실제 존재하는 페이지가 일치하지 않음
- `/instruments` 페이지가 사이드바에 없어서 접근 불가능
- 사장님이 필요한 기능을 찾기 어려움

**해결 방법**:

- ✅ `/instruments` 페이지를 사이드바에 추가
- ✅ `/clients/analytics` 페이지도 사이드바에 추가
- ✅ 경로 매칭 로직 개선 (하위 경로 인식)

**현재 상태**:

- **사이드바 메뉴**: Items, Clients, Connected Clients, Calendar, Sales, **Instruments**, **Client Analytics**
- 모든 주요 페이지가 사이드바에서 접근 가능

**우선순위**: ⭐⭐⭐⭐⭐ (Critical) - **완료**

---

### 🔴 2. 기능 중복 및 혼란

**문제점**:

#### 2.1 Dashboard vs Instruments 페이지

- `/dashboard` (Items): 악기 목록, 추가/수정/삭제
- `/instruments`: 악기 목록, 클라이언트 연결
- 두 페이지가 유사한 기능을 제공하지만 목적이 불명확

**사장님 관점에서의 혼란**:

- "악기를 추가하려면 어느 페이지로 가야 하나?"
- "악기와 클라이언트를 연결하려면 어느 페이지를 사용해야 하나?"

**개선 제안**:

- 두 페이지를 통합하거나
- 각 페이지의 목적을 명확히 구분:
  - **Dashboard**: 악기 재고 관리 및 전체 현황
  - **Instruments**: 악기 상세 정보 및 클라이언트 연결

#### 2.2 Clients 관련 페이지

- `/clients`: 클라이언트 관리 (추가/수정/삭제)
- `/clients/analytics`: 클라이언트별 구매 이력 및 통계
- `/connections`: 클라이언트-악기 연결 관리

**사장님 관점에서의 혼란**:

- "고객 정보를 수정하려면 어느 페이지로 가야 하나?"
- "고객이 어떤 악기를 가지고 있는지 확인하려면 어느 페이지로 가야 하나?"
- "고객의 구매 이력을 보려면 어느 페이지로 가야 하나?"

**개선 제안**:

- `/clients` 페이지에 탭 추가:
  - **정보**: 기본 정보 관리
  - **악기 연결**: 연결된 악기 목록
  - **구매 이력**: 구매 기록 및 통계
- 또는 각 페이지의 목적을 명확히 표시

**우선순위**: ⭐⭐⭐⭐ (High)  
**예상 시간**: 2-3일

## 업무 효율성 개선 (High Priority)

### 🟠 3. 빠른 데이터 입력 개선

**문제점**:

- 악기 추가 시 많은 필드를 입력해야 함
- 반복적인 데이터 입력이 번거로움
- 자주 사용하는 값들을 매번 입력해야 함

**개선 제안**:

#### 3.1 기본값 및 템플릿 기능

- **자주 사용하는 값 저장**: Maker, Type 등 자주 사용하는 값들을 저장하고 드롭다운에서 선택
- **템플릿 기능**: 특정 타입의 악기 추가 시 템플릿을 선택하여 기본값 자동 입력
- **스마트 기본값**: 이전에 입력한 값들을 기억하여 자동 완성

#### 3.2 일괄 입력 기능

- **CSV/Excel 임포트**: 여러 악기를 한 번에 추가
- **복사/붙여넣기**: 다른 시스템에서 데이터를 복사하여 일괄 입력

**우선순위**: ⭐⭐⭐⭐ (High)  
**예상 시간**: 3-5일

---

### ✅ 4. 판매 기록 입력 개선 - **해결 완료**

**문제점**:

- 판매 기록 추가 시 여러 페이지를 이동해야 함
- 악기와 클라이언트를 각각 선택해야 함
- 판매 후 악기 상태 업데이트가 수동으로 필요

**해결 방법**:

#### 4.1 원클릭 판매 기록 - **완료**

- ✅ **Dashboard에서 직접 판매 기록 추가**: 악기 목록의 행 액션 메뉴(⋮)에서 "Sell" 버튼 클릭 시 판매 폼이 열립니다.
- ✅ **자동 상태 업데이트**: 판매 기록 저장 시 악기 상태가 자동으로 "Sold"로 변경됩니다.
- ✅ **초기값 자동 채우기**: 악기 가격, 악기 ID, 연결된 클라이언트 정보가 자동으로 폼에 채워집니다.
- ✅ **빠른 클라이언트 선택**: 클라이언트 목록이 이름 순으로 정렬되어 선택이 용이합니다.

**현재 상태**:

- Dashboard에서 악기 목록의 행 액션 메뉴(⋮) → "Sell" 버튼 클릭
- 판매 폼이 열리며 악기 정보가 자동으로 채워짐
- 판매 기록 저장 시 악기 상태 자동 업데이트

**우선순위**: ⭐⭐⭐⭐ (High) - **완료**

### 🟠 5. 검색 및 필터 개선

**문제점**:

- Serial Number로 검색 시 정확히 일치하는 것만 찾아짐
- 여러 조건으로 필터링하기 어려움
- 필터 상태가 저장되지 않음

**개선 제안**:

#### 5.1 고급 검색 기능

- **부분 일치 검색**: Serial Number의 일부만 입력해도 검색 가능
- **다중 조건 검색**: 여러 필드를 동시에 검색 (예: Maker + Type + Status)
- **저장된 검색**: 자주 사용하는 검색 조건을 저장

#### 5.2 필터 개선

- **필터 상태 저장**: 페이지를 떠났다가 돌아와도 필터 상태 유지
- **빠른 필터**: 자주 사용하는 필터를 빠른 버튼으로 제공
- **필터 조합**: 여러 필터를 조합하여 사용

**우선순위**: ⭐⭐⭐⭐ (High)  
**예상 시간**: 2-3일

### 🟠 6. 캘린더 작업 효율성 개선

**문제점**:

- 유지보수 작업 추가 시 여러 정보를 입력해야 함
- 작업 상태 변경이 번거로움
- 작업과 악기/클라이언트 연결이 불편함

**개선 제안**:

#### 6.1 빠른 작업 추가

- **드래그 앤 드롭**: 캘린더에서 날짜를 드래그하여 작업 추가
- **반복 작업**: 정기적인 작업을 반복 일정으로 설정
- **작업 템플릿**: 자주 사용하는 작업을 템플릿으로 저장

#### 6.2 작업 상태 변경 개선

- **원클릭 상태 변경**: 작업 목록에서 상태를 바로 변경
- **일괄 상태 변경**: 여러 작업을 선택하여 한 번에 상태 변경
- **자동 완료**: 작업 완료 시 다음 단계 자동 제안

**우선순위**: ⭐⭐⭐ (Medium)  
**예상 시간**: 2-3일

## 사용성 개선 (Medium Priority)

### 🟡 7. 대시보드 개선

**문제점**:

- 현재 대시보드가 단순히 악기 목록만 표시
- 중요한 정보를 한눈에 파악하기 어려움
- 빠른 액션 버튼이 부족

**개선 제안**:

#### 7.1 대시보드 위젯 추가

- **요약 카드**: 총 악기 수, 판매 대기 중인 악기, 오늘 예정된 작업 등
- **최근 활동**: 최근 추가된 악기, 최근 판매 기록 등
- **빠른 액션**: 자주 사용하는 기능에 빠르게 접근할 수 있는 버튼

#### 7.2 시각화 개선

- **차트 추가**: 월별 판매 추이, 악기 타입별 분포 등
- **상태별 색상**: 악기 상태에 따라 색상으로 구분
- **우선순위 표시**: 중요한 정보를 눈에 띄게 표시

**우선순위**: ⭐⭐⭐ (Medium)  
**예상 시간**: 3-5일

---

### 🟡 8. 모바일 사용성 개선

**문제점**:

- 모바일에서 데이터 입력이 불편함
- 작은 화면에서 많은 정보를 확인하기 어려움
- 터치 제스처가 제한적

**개선 제안**:

#### 8.1 모바일 최적화

- **터치 친화적 버튼**: 버튼 크기 및 간격 최적화
- **스와이프 제스처**: 목록에서 스와이프로 삭제/수정
- **모바일 전용 레이아웃**: 모바일에 최적화된 레이아웃 제공

#### 8.2 빠른 입력

- **음성 입력**: 악기 정보를 음성으로 입력
- **QR 코드 스캔**: Serial Number를 QR 코드로 스캔하여 입력
- **사진 첨부**: 악기 사진을 바로 첨부

**우선순위**: ⭐⭐⭐ (Medium)  
**예상 시간**: 3-5일

---

### 🟡 9. 데이터 확인 및 수정 개선

**문제점**:

- 악기 정보를 수정하려면 여러 단계를 거쳐야 함
- 정보를 확인하려면 상세 페이지로 이동해야 함
- 인라인 편집이 제한적

**개선 제안**:

#### 9.1 인라인 편집 개선

- **더 많은 필드 인라인 편집**: 가격, 상태 등 자주 변경하는 필드를 인라인으로 편집
- **즉시 저장**: 변경 사항을 즉시 저장
- **변경 이력**: 변경 이력을 추적하여 필요시 되돌리기

#### 9.2 상세 정보 표시

- **호버 카드**: 마우스를 올리면 상세 정보를 빠르게 확인
- **모달 개선**: 상세 정보를 모달로 표시하여 페이지 이동 없이 확인
- **빠른 액션**: 상세 정보에서 바로 수정/삭제 가능

**우선순위**: ⭐⭐⭐ (Medium)  
**예상 시간**: 2-3일

#### 10.2 알림 설정

- **알림 설정 페이지**: 알림 종류 및 빈도 설정
- **알림 필터**: 원하는 알림만 받기
- **알림 기록**: 알림 이력 확인

**우선순위**: ⭐⭐⭐ (Medium)  
**예상 시간**: 2-3일

- 모든 페이지에 일관된 Empty State 컴포넌트 사용
- 사용자 경험 일관성 향상

### 3. 에러 메시지 표시 방식 통일 (1-2일)

- 에러 타입에 따른 표시 방식 명확화
- 사용자 혼란 감소
