### 방안 2: 명확한 차별화

**Dashboard (`/dashboard`)**:

- **목적**: 빠른 재고 관리 및 판매
- **기능**:
  - 인라인 편집 (상태, 가격)
  - 판매 기능
  - 빠른 필터링
  - 클라이언트 연결 정보 표시
- **사용자**: 일일 운영, 빠른 작업

**Instruments (레거시 `/instruments` 경로)**:

- **목적**: 악기 상세 정보 관리 및 카탈로그
- **기능**:
  - 상세 정보 편집 (모달 또는 별도 페이지)
  - 이미지 관리
  - 인증서 관리
  - 상세 메모
  - 클라이언트 연결 관리 (상세)
- **사용자**: 상세 정보 입력, 카탈로그 관리

**작업 내용**:

1. Instruments 페이지에 편집/삭제 기능 추가
2. Instruments에 상세 정보 관리 기능 강화
3. Dashboard는 빠른 작업 중심으로 유지
4. 각 페이지의 목적을 UI/문서로 명확히 표시

**예상 시간**: 3-4일

**현재 상태**: ⏸️ 해당 옵션은 페이지 통합 이후 유효하지 않음 (Instruments 페이지가 제거됨)
